MINEFLEX

SERVER CREATION PROCESS

Mineflex is a Minecraft Server Hosting platform that operates on a cloud-computing model

The server signup process is the users first experience as a Mineflex user. Once a beta customer has signed up through the Landing Page they are directed to make an account and then create a server. Creating a Minecraft server requires important steps that allow the user to customize their server to suit individual needs. The focus of the design was to streamline the process through appropriate categorical grouping and a simple interface.

THE CHALLENGE

The biggest chunk of time was spent breaking-down the original server creation process. With each step previously having its own page, I needed to find a way to re-organize the process. This meant combining steps while maintaining an easy-to-use, uncluttered interface.

An unexpected challenge was the server list page (step one) and finding the best placement and design for the "Add Server" Button. Another important feature missing in the early design was a progress bar. There were too many steps in the original design and there was no indicator to let the user know where they were in the process.

OUR APPROACH

There were originally nine steps which were reduced down to four in the V2 design.

I kept the original idea of cards for the steps with categorial options and added a progress bar that updated as the user went through each step rather than when they changed pages.

THE DETAILS

SERVER LIST

The server list page, although simple, required more attention because it is effectively the home page for an existing user. From the server list a user can make changes to their current server(s) or add a new one.

Features

Small changes to key features made a huge difference on the server list page. Its inherent simplicity allowed design details like the server state, add server button and overall page setup to stand well on its own.

The server state button colors were pulled from the color palette used in the rest of the control panel. Accessibility drove a lot of the color decisions I made, with special attention paid to areas where there was a monochromatic color scheme. I used the online Oracle tool to be sure colorblindness would not effect an idividual's ability to use the product.

To give the design continuity from the control panel I replaced the beta select buttons with whole element selectors. The color and border resemble the "Add Server" button that navigates a user to the first step in the process and gave a natural flow from one page the next.

The "+" sign we used during the beta didn't work as well with the later designs. Instead I used a button design from the control panel and moved it to the top right of the page so it didn't sit awkwardly on the page.
