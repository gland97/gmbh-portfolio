MINEFLEX

CONTROL PANEL REDESIGN

Mineflex is a Minecraft Server Hosting platform that operates on a cloud-computing model

The central goal of Mineflex is to provide Minecraft players with a server host that bills by the hour. Unlike traditional server hosts that bill monthly, Mineflex operates on a cloud-computing model, meaning users only pay for what they use each month. In addition, Mineflex is dedicated to providing a custom control panel built specifically for Minecraft users. The Control Panel redesign grew from user interest and the large adoption of the service during the Beta period from September 2020 to January 2021.

THE CHALLENGE

We launched the Mineflex Beta in September 2020 to gauge market interest in a Minecraft Server Host that billed hourly instead of monthly.

After 500+ email signups by December and 150+ members providing feedback in the Mineflex Discord channel we aimed to develop a Mineflex V2. The Mineflex Control Panel was the defining aspect of the product and needed an entirley new UI/UX design.

OUR APPROACH

The Mineflex Discord Channel was essential to gathering feedback during the beta. Delving deep into market research and organizing user feedback drove the prioritization of features for the Mineflex V2 MVP.

Every design decision came back to the central question:

Does this design serve a user who is new to Minecraft server hosting while providing advanced features for those who are not?

THE DETAILS

File Manager

The file manager was the most complex aspect of the control panel. File management needed to span across four separate categories which we ultimatley decided to combine into a “Files and Configuration” page.

File Manager for Files and Configuration: Plugins

Iterations

After early wireframes and dicussions with the engineer we decided on a side navigation panel which would split each section into its own page. The style helped organize needs while helping the interface be easily navigable.

Users requested being able to easily switch between file managers for each section. This drove the decision to create one page for "Files and Configuration" while splitting up sections using tabs on the same page.

Another concept we had was providing quick upload buttons for user accesibility. We decided ultimatley that the interface should be easy enough to use without a quick upload button and moved the idea into the backlog for later development.

The final design allows the user to see jar files and configuration files simultaneously. We also added convenient features like bulk delete and download options.

One of my favorite features of the file manager is the upload module. Initial designs had the drag-and-drop feature within the file manager page. As the file manager became more detailed we implemented an "Add" button which then promts the Upload Module.

The change in design meant there was more space to offer a complete "upload status" area for current and previous downloads while reserving plenty of space for the drag-and-drop area we didn't want to lose.

Backup Page

Designing the backup page was research-heavy. The biggest callenge was understanding its funtion in the context of how Mineflex billing sysytem is set up and arranging the features on the page without overcrowding it.

Iterations

A large focus of the Mineflex V2 Backup Page was including the usage summary and billing within the page. We wanted a user to be able to see their prior usage and edit their backup preferences all in one place, rather than having them navigate to another internal page.

The first iteration took features and design inspiration from mimicking the file manager while including more obvious bulk actions: Restore, Download and Delete.

Later designs began to implement ways to customize a user's server backup plan. I added toggle bars and tool-tips. Once the features we wanted to include were added onto the page I then began thinking about the most effective way to organize them.

To clean up the top half of the backups page I grouped the content into cards and used a drop-down, instead of tabs for backup options in-step with the other pages. A final decision was adding four cards that gave an easy-to-digest summary of a user's current backup usage.
